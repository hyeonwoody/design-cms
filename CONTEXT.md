# Design System CMS

Build a design-token CMS for UI/UX designers through a five-stage process: from
requirements through a working, documented Django application.

## Task Routing

| Task Type | Go To | Description |
|-----------|-------|-------------|
| Define requirements | `stages/01-research/CONTEXT.md` | Brief into variable/fixed data split + tab inventory |
| Design app structure | `stages/02-discovery/CONTEXT.md` | App boundaries, page vs API routes, tab model |
| Model the domain | `stages/03-domain-modeling/CONTEXT.md` | Django models, migrations, ERD |
| Build app + API + CMS | `stages/04-core-api/CONTEXT.md` | MVT pages, DRF API, Admin CMS, Figma import |
| Write docs | `stages/05-documentation/CONTEXT.md` | README, architecture, .env.example |

## Shared Resources

| Resource | Location | Contains |
|----------|----------|----------|
| Domain context | `design-system/CONTEXT.md` | Routes to the token contract and tab model |
| Shared files | `shared/` | Code constants and seed-data plan used across stages |
